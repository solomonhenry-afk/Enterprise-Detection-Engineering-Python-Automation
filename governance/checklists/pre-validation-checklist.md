# Pre-Validation Checklist

**Document ID:** LHT-Governance-Checklist-002

---

# Purpose

This checklist must be completed before any security validation, Purple Team exercise, attack-path validation, or defensive testing is performed.

---

# Assessment Objective

- [ ] Validation objective documented.
- [ ] Technique identified.
- [ ] MITRE ATT&CK mapping completed.
- [ ] Expected outcome documented.

---

# Scope Verification

- [ ] Target systems verified.
- [ ] Test accounts identified.
- [ ] Tier Zero assets reviewed.
- [ ] Authorization confirmed.

---

# Detection Readiness

- [ ] Splunk detection available.
- [ ] Sysmon configured.
- [ ] Windows Event IDs documented.
- [ ] Detection owner identified.

---

# Evidence Preparation

- [ ] Evidence IDs assigned.
- [ ] Screenshot locations prepared.
- [ ] Validation workbook updated.
- [ ] Evidence storage verified.

---

# Operational Safety

- [ ] Stop conditions documented.
- [ ] Rollback plan available.
- [ ] Sensitive accounts excluded.
- [ ] Business impact assessed.
