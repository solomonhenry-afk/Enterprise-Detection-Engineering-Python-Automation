---

# Purpose

This checklist ensures that all prerequisites have been satisfied before collecting Active Directory security assessment evidence. The objective is to maintain evidence integrity, authorization compliance, and data quality throughout the assessment.

---

# Authorization

- [ ] Assessment scope has been formally defined.
- [ ] Authorization boundaries have been reviewed.
- [ ] Rules of Engagement (RoE) have been acknowledged.
- [ ] Assessment activities are limited to the authorized Lighthouse Technology lab environment.
- [ ] No production systems are included in scope.

---

# Environment Readiness

- [ ] Domain Controllers are reachable.
- [ ] Administrative workstation is prepared.
- [ ] Required PowerShell modules are available.
- [ ] Required assessment tools are installed.
- [ ] Time synchronization is verified.
- [ ] Network connectivity has been validated.

---

# Evidence Repository

- [ ] Evidence folders exist.
- [ ] Raw evidence directory is prepared.
- [ ] Normalized evidence directory is prepared.
- [ ] Screenshot directory exists.
- [ ] Evidence manifest template is available.
- [ ] Naming conventions have been reviewed.

---

# Logging & Telemetry

- [ ] Windows Event Logging enabled.
- [ ] Sysmon operational.
- [ ] Splunk receiving telemetry.
- [ ] Time stamps synchronized.
- [ ] Event retention verified.

---

# Data Quality

- [ ] Collection tools validated.
- [ ] Duplicate collection avoided.
- [ ] Expected evidence outputs documented.
- [ ] Collection methodology recorded.

---

# Risk Controls

- [ ] Destructive testing excluded.
- [ ] Credential-impacting activities reviewed.
- [ ] Rollback procedures documented.
- [ ] Collection impact assessed.
