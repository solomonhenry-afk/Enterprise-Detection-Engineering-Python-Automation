# Purpose

This document establishes publication standards for all public portfolio artifacts produced during the Enterprise Security Evolution program.

The objective is to ensure published work is technically accurate, ethically responsible, evidence-backed, and suitable for recruiters, hiring managers, technical reviewers, and executive audiences.

---

# Publication Principles

Every published artifact shall be:

- Evidence-led
- Technically accurate
- Governance-aligned
- Ethically responsible
- Clearly scoped
- Recruiter-friendly
- Reproducible where appropriate

---

# Public Content Requirements

The following information may be published:

- Assessment methodology
- Security architecture
- Detection engineering
- MITRE ATT&CK mappings
- Governance documentation
- Remediation recommendations
- Normalized sample datasets
- Simulated Purple Team validation
- Executive reporting examples

---

# Information That Shall Not Be Published

The following must never appear in the public repository:

- Production credentials
- Real customer data
- Personally identifiable information (PII)
- Private cryptographic material
- Active secrets or API keys
- Internal IP addresses from non-lab environments
- Sensitive configuration files
- Proprietary customer documentation

# Reporting Standards

Every report shall:

- Reference supporting evidence.
- Distinguish proposed, simulated, collected, validated, and remediated artifacts.
- Avoid unsupported technical claims.
- Include remediation guidance where appropriate.
- Clearly define assessment scope and authorization boundaries.

---

# GitHub Publication Checklist

Before publishing:

- [ ] Repository reviewed.
- [ ] Sensitive data removed.

- [ ] README updated.
- [ ] Evidence classifications verified.
- [ ] Documentation reviewed.

# LinkedIn Publication Checklist

Before posting:

- [ ] Technical summary reviewed.
- [ ] Executive summary prepared.
- [ ] Screenshots verified.
- [ ] No sensitive information disclosed.
- [ ] GitHub repository linked.
- [ ] Portfolio branding applied.

# Publication Statement

All work contained within this repository was performed exclusively within authorized laboratory environments owned, managed, or explicitly authorized by Lighthouse Technology. Where techniques involve offensive security testing, the portfolio emphasizes controlled validation, evidence collection, detection engineering, and remediation guidance rather than operational misuse.
