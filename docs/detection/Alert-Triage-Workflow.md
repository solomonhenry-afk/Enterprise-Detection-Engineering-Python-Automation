# 1. Purpose

This document defines the standardized workflow for triaging security alerts generated by SIEM, IDS/IPS, EDR, and other security monitoring technologies within the Lighthouse Technology Enterprise Security Evolution program.

The objective is to ensure alerts are assessed consistently, enriched with supporting evidence, prioritized based on business risk, and escalated appropriately while minimizing analyst fatigue and reducing false positives.

---

# 2. Objectives

The Alert Triage Workflow enables:

- Consistent analyst decision-making
- Faster identification of true security incidents
- Risk-based alert prioritization
- Structured evidence collection
- Reduced Mean Time to Detect (MTTD)
- Improved Mean Time to Respond (MTTR)
- Detection quality improvement through feedback

# 9. Alert Classification

Classify alerts as:

True Positive

Verified malicious activity requiring response.

False Positive

Benign activity incorrectly identified.

Benign Positive

Expected administrative activity.

Needs Investigation

Insufficient evidence to determine outcome.

Duplicate

Already investigated.

Suppressed

Covered by an approved suppression rule.

# 11. Investigation Decision

If True Positive

Open incident

Notify stakeholders

Contain threat

Collect forensic evidence

Initiate response procedures

---

If False Positive

Document reason

Record affected rule

Recommend tuning

Update detection documentation

---

If Needs Investigation

Escalate to Tier 2 or Tier 3 analyst

Perform additional log analysis

Correlate with other alerts

Expand timeline

---

# 12. Escalation Criteria

Escalate when:

Privilege escalation detected

Tier Zero assets affected

Domain Controller targeted

Multiple systems involved

Evidence of lateral movement

Credential compromise suspected

Persistence identified

Data exfiltration suspected

Ransomware indicators present

Critical business systems affected
