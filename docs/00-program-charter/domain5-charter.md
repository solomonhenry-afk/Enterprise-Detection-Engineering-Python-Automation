# Domain 5 Charter — Enterprise Active Directory Penetration Testing & Security Auditing

## Program Identity

**Program:** Enterprise Security Evolution — Domain 5
**Organization:** Lighthouse Technology
**Purpose:** Perform an evidence-backed Active Directory security assessment that combines auditing, identity enumeration, attack-path analysis, controlled adversary validation, detection engineering, remediation planning, and executive risk communication.

## Strategic Objective

Identify, validate, prioritize, and reduce identity-based risk within the authorized Lighthouse Technology Active Directory lab environment.

## Scope

The assessment includes Active Directory forest and domain configuration, domain controllers, Group Policy, organizational units, DNS, SMB, LDAP, Kerberos, trusts, privileged identities, service accounts, delegation, Tier Zero assets, Windows telemetry, Sysmon telemetry, and Splunk detection coverage.

## Program Outcomes

* Enterprise Active Directory Security Assessment Report
* Executive Risk Summary
* Attack Path Analysis
* Privilege Exposure Report
* Detection Coverage Matrix
* MITRE ATT&CK Matrix
* Purple Team Validation Report
* Active Directory Security Audit Report
* Enterprise Remediation Roadmap
* Executive Presentation
* Evidence Manifest and Data Quality Report
* GitHub-ready repository and portfolio package

## Evidence Claim Standard

All project claims must be labeled as one of the following:

* **Proposed:** Planned activity or expected outcome not yet tested.
* **Simulated:** Controlled test data or lab scenario created for validation.
* **Collected:** Evidence acquired directly from the authorized lab.
* **Validated:** Finding or detection confirmed through approved testing.
* **Remediated:** Corrective action implemented and documented.
* **Retested:** Post-remediation validation completed.

## Cross-Domain Integration

Domain 5 consumes telemetry and detection engineering from Domain 1, controlled adversary-validation practices from Domain 2, network exposure and infrastructure context from Domain 3, and governance, evidence lineage, risk scoring, and executive reporting from Domain 4.
